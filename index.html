GitBashでユーザー登録 2020-03-20|Windows-Git

GitのCUIツールであるGitBashの操作方法の解説

ディレクトリについて

CUIではディレクトリと言う言葉が登場する。
ディレクトリとはフォルダーと同じ意味である。

ディレクトリの種類

ディレクトリには三つの種類がある。

  • ルートディレクトリ
    一番階層が浅い
  • ホームディレクトリ
    Git Bash等を起動したときに初期表示されるディレクトリ
  • カレントディレクトリ
    現在のディレクトリ

ディレクトリの図

コラム

三つのディレクトリには、特別なパスの書き方が設定されている。

二種類のパス

  • 絶対パス
    ルートディレクトリを起点としたパス

  • 相対パス
    カレントディレクトリを起点としたパス

ディレクトリを操作するコマンド

Gitのコマンドではない

  • pwd
    カレントディレクトリの絶対パスが出力される
  • mkdir
    新しいディレクトリを作成する
  • ls
    ディレクトリの内容を確認する
  • cd
    カレントディレクトリを移動する

コラム

Git Bashではmkdirを使用してディレクトリを作成するが、
Windowsのコマンドプロンプトではmd指定だけでもディレクトリの作成が出来る。

Git BashでGitの設定

Gitの設定をするにはgit configと言うコマンドを利用する。
–globalオプションを付けると、
ホームディレクトリの.gitconfigファイルに設定が反映される。

ユーザー名を設定

git config –global user.name 名前入力

メールアドレスの登録

git config –global user.email アドレス入力

設定内容の確認

git config –list

設定内容を確認するコマンドを入力すると、
user.name、user.emailの項目に先ほど入力したものが、反映されたか確認出来る。

エディタの設定

git config –global core.editor “エディタのファイルパス –wait”

Gitの使用中にメッセージを入力する事がある。
その時に使いたいエディタがある場合はこのコマンドで設定出来る。

git config core.editor
上記のコマンドでエディタの登録が出来たかが確認出来る。

コラム

プロキシ環境でネットに接続する場合、
リモートリポジトリと接続出来ない問題が発生する事がある。
その場合はGitにプロキシサーバーの設定をする必要がある。

プロキシ環境でGitを利用する場合
git config –global http.proxy プロキシサーバーURL:ポート番号

プロキシ認証が必要な場合
git config –global http.proxy プロキシのユーザー名：パスワード@プロキシサーバーURL:ポート番号

アドレス同士の計算 2020-02-14|C++

変数を宣言すると、型に合わせてアドレスが確保される。
確保されたアドレスを使用して計算する事が出来る。

説明

n配列の先頭アドレスを100とする。

pはn配列の先頭アドレスが入る為、100のアドレスが代入される。
その為、始めのcoutでは同じ値が出力される。
次のp+=2;ではint型の2を代入している。(8バイト分)
その後のcoutでは2が加算されたpとn配列のn[2]を出力する。
同じ値をさしている事が確認出来る。

最後にアドレス同士の計算。
p - n の計算を分かりやすくすると下記の通り。
p(108) - n(100) = 8
8バイト分足したpとnの先頭アドレスの差は8になる。

しかし、出力すると結果は2と表示される。
これはポインタ同士で計算をすると、値を求めた後にポインタが指している変数の型で割った結果を出す様にコンパイラが動く為である。
