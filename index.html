文字列の操作 2020-09-14|C++

文字列の操作

stringの使い方

文字列の代入、足しこみ、出力

文字の関数表

char& operator[](size_type i);
const char& operator[](size_type i)const;
i番目の要素への参照を返す。iの範囲はチェックしない。




char& at(size_type i);
const char& at(size_type i)const;
i番目の要素への参照を返す。iが有効なインデックスの範囲外の場合、out_of_range例外が投げられます。




size_type size()const;
size_type length()const;
文字列の長さ(バイト数)を返す。




void resize(size_type n);
文字列のサイズを変更します。




void reserve(size_type n = 0);
内部で動的に確保されるメモリをあらかじめ最低でもnだけ確保しておく。




size_type capacity()const;
内部で動的に確保されているメモリサイズを返す。




void clear();
文字列を空にする。




bool empty()const;
文字列が空かどうかを返す。




string& insert(size_type pos = 0, const char* s);
string& insert(size_type pos = 0, const string& s);
pos 文字目の位置にsを挿入します。




string& erase(size_type pos = 0, size_type n = npos);
pos文字目からn文字削除し、間を詰めます。nがnposの場合には最後まで削除します。




string& replace(size_type pos, size_type n, const char* s);
string& replace(size_type pos, size_type n, const string& s);
pos文字目からn文字をsで置き換えます。




size_type find(const char* s, size_type pos = 0)const;
size_type find(const string& s, size_type pos = 0)const;
pos文字目から検索し、sが見つかった地点を返す。。見つからなければnposを返す。




size_type rfind(const char* s, size_type pos = npos)const;
size_type rfind(const string& s, size_type pos = npos)const;
pos文字目から逆向きに検索し、sが見つかった地点を返す。見つからなければnposを返す。
posがnposの場合には文字列の終端から検索を開始します。




string substr(size_type pos = 0, size_type n = npos);
pos文字目からn文字の文字列を持つstringクラスのオブジェクトを作って返す。
nがnposの場合にはpos文字目から最後までの文字列を持つstringクラスのオブジェクトを作って返す。




const char* c_str()const;
ヌル文字のついた文字列アドレスを返す。




const char* data()const;
文字列のアドレスを返す。ヌル文字は無い。

GitBashでユーザー登録 2020-03-20|Git-Windows

GitのCUIツールであるGitBashの操作方法の解説

ディレクトリについて

CUIではディレクトリと言う言葉が登場する。
ディレクトリとはフォルダーと同じ意味である。

ディレクトリの種類

ディレクトリには三つの種類がある。

  • ルートディレクトリ
    一番階層が浅い
  • ホームディレクトリ
    Git Bash等を起動したときに初期表示されるディレクトリ
  • カレントディレクトリ
    現在のディレクトリ

ディレクトリの図

コラム

三つのディレクトリには、特別なパスの書き方が設定されている。

二種類のパス

  • 絶対パス
    ルートディレクトリを起点としたパス

  • 相対パス
    カレントディレクトリを起点としたパス

ディレクトリを操作するコマンド

Gitのコマンドではない

  • pwd
    カレントディレクトリの絶対パスが出力される
  • mkdir
    新しいディレクトリを作成する
  • ls
    ディレクトリの内容を確認する
  • cd
    カレントディレクトリを移動する

コラム

Git Bashではmkdirを使用してディレクトリを作成するが、
Windowsのコマンドプロンプトではmd指定だけでもディレクトリの作成が出来る。

Git BashでGitの設定

Gitの設定をするにはgit configと言うコマンドを利用する。
–globalオプションを付けると、
ホームディレクトリの.gitconfigファイルに設定が反映される。

ユーザー名を設定

git config –global user.name 名前入力

メールアドレスの登録

git config –global user.email アドレス入力

設定内容の確認

git config –list

設定内容を確認するコマンドを入力すると、
user.name、user.emailの項目に先ほど入力したものが、反映されたか確認出来る。

エディタの設定

git config –global core.editor “エディタのファイルパス –wait”

Gitの使用中にメッセージを入力する事がある。
その時に使いたいエディタがある場合はこのコマンドで設定出来る。

git config core.editor
上記のコマンドでエディタの登録が出来たかが確認出来る。

コラム

プロキシ環境でネットに接続する場合、
リモートリポジトリと接続出来ない問題が発生する事がある。
その場合はGitにプロキシサーバーの設定をする必要がある。

プロキシ環境でGitを利用する場合
git config –global http.proxy プロキシサーバーURL:ポート番号

プロキシ認証が必要な場合
git config –global http.proxy プロキシのユーザー名：パスワード@プロキシサーバーURL:ポート番号

アドレス同士の計算 2020-02-14|C++

変数を宣言すると、型に合わせてアドレスが確保される。
確保されたアドレスを使用して計算する事が出来る。

説明

n配列の先頭アドレスを100とする。

pはn配列の先頭アドレスが入る為、100のアドレスが代入される。
その為、始めのcoutでは同じ値が出力される。
次のp+=2;ではint型の2を代入している。(8バイト分)
その後のcoutでは2が加算されたpとn配列のn[2]を出力する。
同じ値をさしている事が確認出来る。

最後にアドレス同士の計算。
p - n の計算を分かりやすくすると下記の通り。
p(108) - n(100) = 8
8バイト分足したpとnの先頭アドレスの差は8になる。

しかし、出力すると結果は2と表示される。
これはポインタ同士で計算をすると、値を求めた後にポインタが指している変数の型で割った結果を出す様にコンパイラが動く為である。
